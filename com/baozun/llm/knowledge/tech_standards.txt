技术部门规范
============

开发语言和框架：
- 后端：Java (Spring Boot), Python (FastAPI)
- 前端：React, Vue.js
- 数据库：MySQL, Redis, MongoDB
- 消息队列：RabbitMQ, Kafka
- 容器化：Docker + Kubernetes

代码规范：
1. 所有代码必须通过 Code Review 才能合并到主分支
2. 单元测试覆盖率不低于 80%
3. 提交信息格式：[类型] 简短描述（如 [feat] 添加用户登录功能）
4. 分支命名：feature/xxx, bugfix/xxx, hotfix/xxx

部署流程：
- 开发环境：提交代码后自动部署
- 测试环境：通过 CI 流水线自动部署，需全部测试通过
- 预发环境：需要 Tech Lead 审批
- 生产环境：需要 Tech Lead + 产品经理 双重审批，只在每周二和周四发布

故障处理：
- P0 故障（系统完全不可用）：15分钟内响应，1小时内恢复
- P1 故障（核心功能受损）：30分钟内响应，4小时内恢复
- P2 故障（非核心功能异常）：2小时内响应，24小时内修复

值班制度：
- 每周轮换，7x24小时待命
- 值班人员需保持手机畅通
- 值班补贴：工作日 200元/天，周末 400元/天
